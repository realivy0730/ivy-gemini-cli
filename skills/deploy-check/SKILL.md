# Deploy Check (部署前檢查)

## 架構描述
部署前自動檢查清單，驗證配置、安全性、備份、回滾方案，確保安全部署。

## 觸發方式
- "部署前檢查 [環境]"
- "deploy check [service]"

## 嚴格執行規範
- 不涉及腳本生成，僅執行檢查與驗證
- 自動建立 TODO 清單，每步驟強制驗收
- 遇異常立即暫停，執行 RCA
- 生產環境強制二次確認、備份驗證、回滾方案檢查

## 部署前評估問題（Assessment Questions 模式）

部署前先釐清以下問題，再進入檢查清單：

1. 這次部署影響哪些服務和使用者？
2. 是否有資料庫 schema 變更？需要 migration 嗎？
3. 回滾方案是什麼？需要多久？
4. 是否有相依服務需要同步更新？
5. 監控和告警是否已設定？

## 執行流程

### 前置檢查
- [ ] 確認目標環境（dev/staging/production）
- [ ] 若為 production，要求明確確認

### 主要步驟
1. **檢查配置** — 驗證 YAML/JSON 語法、環境變數、版本號
2. **對抗性審查 (Review Gate)** — **[強制執行]** 呼叫 `code-adversary` 技能進行評分
   - **通過條件**：分數 >= 90
   - **阻斷條件**：分數 < 70 (立即終止部署)
   - **警告條件**：70 <= 分數 < 90 (需人工確認)
3. **驗證備份** — 最新備份時間 < 24h、備份可還原
4. **檢查回滾方案** — 回滾腳本存在、步驟已測試
5. **安全性掃描** — 無硬編碼密鑰、SSL 有效、防火牆正確
6. **監控檢查** — 監控已設定、告警已配置
7. **生成報告** — 含所有檢查項目、阻斷項目為 0

## 生產環境保護（最高優先級）

### 阻斷項目（禁止部署）
- 無備份或備份過期（> 24h）、無回滾方案
- 硬編碼密鑰、SSL 憑證已過期、監控未設定

### 警告項目（建議修正）
- SSL 憑證即將過期（< 30 天）、備份時間 > 12h

## 驗收標準
- [ ] 所有檢查項目已執行、阻斷項目為 0
- [ ] Review Gate 評分已達標 (>= 90)
- [ ] 備份已驗證、回滾方案已確認
- [ ] 報告已生成

詳細 workflow 配置參見 `references/workflow-config.md`。

## 注意事項
- 阻斷項目必須修正才能部署
- 每次部署都必須執行完整檢查
- 異常時立即暫停並執行 RCA
