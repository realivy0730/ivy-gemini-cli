# Google Cloud 完善架構框架 (WAF) —— 安全性支柱

## 概覽

Google Cloud 完善架構框架的安全性支柱提供了設計原則與最佳實踐，透過將安全性整合到雲端工作負載架構的每一層，構建強大的安全狀態。它專注於維護數據與系統的機密性與完整性，同時確保合規性與隱私。

## 核心原則

安全性支柱的建議符合以下核心原則：

-  **實施設計安全 (Security by Design)**：從初始設計階段就整合雲端安全與網路安全考量。
-  **實施零信任 (Zero Trust)**：採用「永不信任，始終驗證」的方法，根據持續的信任驗證授予存取權。
-  **實施左移安全 (Shift-left Security)**：在軟體開發生命週期的早期實施安全控制。
-  **實施先發制人的網路防禦**：採取主動的安全方法，如利用威脅情資。
-  **安全且負責任地使用 AI**：以負責任且安全的方式開發和部署 AI 系統。
-  **將 AI 用於安全**：利用 AI 能力（如 Gemini in Security）改進現有的安全系統與流程。
-  **滿足監管、合規與隱私需求**：遵守特定產業的監管與合規標準。

## 相關 Google Cloud 產品

以下是與安全性相關的 Google Cloud 產品與功能範例：

- **身分與存取管理**：
  - **IAM**：對資源的細粒度存取控制。
  - **IAP**：無需 VPN 即可安全存取應用程式。
- **網路安全**：
  - **Google Cloud Armor**：DDoS 保護與 Web 應用程式防火牆 (WAF)。
  - **VPC Service Controls**：定義安全邊界以防止數據外洩。
  - **Cloud NGFW**：進階網路流量威脅保護。
- **數據安全**：
  - **Cloud KMS**：管理加密金鑰。
  - **敏感數據保護 (Sensitive Data Protection)**：發現並脫敏敏感數據。
  - **機密運算 (Confidential Computing)**：加密使用中（記憶體）的數據。
- **安全維運 (SecOps)**：
  - **Security Command Center (SCC)**：集中式的漏洞與威脅管理。
  - **Cloud Logging & Monitoring**：系統活動的可見性。

## 工作負載評估問題

提出適當的問題以了解工作負載的安全性需求：

- 您如何確保安全性整合到您的開發生命週期中？
- 您如何實施存取控制的最小權限原則？
- 您如何在 Google Cloud 環境中保護傳輸中與靜態數據？
- 您如何主動識別並緩解潛在的安全威脅？
- 您如何確保 AI 模型與數據的安全性？

## 驗證清單 (Validation Checklist)

使用以下清單評估架構與安全性建議的一致性：

- **設計安全**：根據安全特性與硬化情況選擇系統組件。
- **零信任**：基於使用者身分與上下文（設備、位置）強制執行存取控制。
- **左移安全**：基礎設施使用 Terraform 等 IaC 工具進行佈署，且 CI/CD 管道整合了自動化安全掃描。
- **先發制人防禦**：所有關鍵資源都啟用了集中式的安全日誌記錄。
- **AI 安全**：AI 管道具備防止篡改與數據中毒的安全措施。
