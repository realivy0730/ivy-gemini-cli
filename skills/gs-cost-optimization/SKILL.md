# Google Cloud 完善架構框架 (WAF) —— 成本優化支柱

## 概覽

Google Cloud 完善架構框架的成本優化支柱提供了一種結構化方法，在最大化業務價值的同時優化雲端工作負載的成本。雲端成本與地端資本支出 (CapEx) 模型顯著不同，需要轉向營運支出 (OpEx) 管理與責任制文化 (FinOps)。

## 核心原則

成本優化支柱的建議符合以下核心原則：

-  **將雲端支出與業務價值對齊**：確保您的雲端資源通過將 IT 支出與業務目標對齊來提供可衡量的業務價值。
-  **培育成本意識文化**：確保組織各部門的人員在決策和活動中考慮成本影響。
-  **優化資源使用**：僅配置所需的資源，並僅為您消耗的部分付費。
-  **持續優化**：持續監控雲端資源使用情況與成本，並根據需要主動調整以優化支出。

## 相關 Google Cloud 產品

以下是與成本優化相關的 Google Cloud 產品與功能範例：

- **可見性與監控**：
  - **Cloud Billing 報表**：用於可視化支出與趨勢的原生儀表板。
  - **BigQuery 帳單匯出**：使用 SQL 與 BI 工具對帳單數據進行細粒度、自定義的分析。
  - **帳單告警與預算**：當支出達到預定義閾值時的自動通知。

- **自動化與優化工具**：
  - **推薦工具 (Recommender) / Active Assist**：自動識別閒置資源、調整權利機會以及未使用的承諾。
  - **FinOps 中心 (FinOps hub)**：在一個儀表板中呈現活動的節省與優化機會。

- **高效基礎設施**：
  - **受管服務與無伺服器服務**：如 Cloud Run 與 GKE Autopilot，減少營運開銷並實現按需付費擴充。
  - **Compute Engine**：針對容錯工作負載使用 Spot VMs，針對穩定工作負載使用承諾使用折扣 (CUDs)。
  - **Cloud Storage 生命週期政策**：根據保留時間或存取頻率自動將數據移至低成本存儲級別。

## 工作負載評估問題

提出適當的問題以了解工作負載的成本相關需求與約束：

- 您如何在雲端架構設計過程中納入成本考量？
- 您如何優化運算資源的成本？
- 您如何平衡成本優化與敏捷創新之間的需求？
- 您如何確保沒有過度配置雲端資源？
- 您如何衡量雲端成本優化倡議的成功？

## 驗證清單 (Validation Checklist)

使用以下清單評估架構與成本優化建議的一致性：

- **成本歸屬**：100% 的資源都標有標籤（例如 `env`, `team`, `app`）。
- **細粒度可見性**：已啟用 BigQuery 帳單匯出，並用於定期成本審查。
- **預算與告警**：每個專案或業務部門都有定義預算與活動告警。
- **權利調整 (Rightsizing)**：根據 Active Assist 提供建議定期調整資源。
- **承諾策略**：每月審查支出以優化承諾使用折扣 (CUD) 覆蓋範圍。
- **受管服務**：除非有特定技術約束，否則新工作負載優先選擇無伺服器選項。
- **存取層級**：為所有主要 Storage Bucket 啟用生命週期政策以最小化封存成本。
